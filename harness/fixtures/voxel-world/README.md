# Canonical voxel-world fixture

The single shared, abstract voxel world that the launchable-voxel tasks (meshing /
render projection, picking, editing, save/replay, smoke) build on instead of ad-hoc
parallel test worlds (launchable-voxel-03, #2434).

All files here are **generated** by the `fixture-maker` tool — none are
hand-authored. The generator (`engine-rs/crates/tools/fixture-maker`) is the source
of truth; this directory is its committed output.

## Contents

| File | Generated by | Format |
|---|---|---|
| `voxel-world.manifest.json` | `fixture_maker::render_fixture` | small fixture index: grid spec, materials, per-chunk table (coord, material, artifact, chunk + content hashes), world hash |
| `chunk_<x>_<y>_<z>.snapshot` | `fixture_maker::build_world` → `rule-voxel-edit::persist::encode_chunk_snapshot` | RLE chunk snapshot (reused persist format, not duplicated) |

## Shape

- One grid (`id 1`, voxel size `1.0`, cubic chunk dims `2×2×2` taken from the grid
  spec — not a hardcoded global chunk size).
- A `2×2×1` arrangement of chunks (4 chunks). Each chunk's bottom layer (`z=0`) is
  solid, so solids meet across every shared X/Y face — exercising chunk borders and
  neighbour invalidation — while the top layer stays empty so exposed faces remain.
- Three abstract materials (ids `1, 2, 3`), validated against a `MaterialCatalog`.
  No product-domain nouns.

## Related generated interaction fixture

`harness/fixtures/voxel-interaction/basic-selection.json` reuses this canonical
world for the basic graphical voxel interaction proof. It records deterministic
camera/selection/edit-anchor evidence generated by the same `fixture-maker` CLI.

## Regenerate / verify

```bash
cd engine-rs
cargo run -p fixture-maker -- write    # regenerate this directory
cargo run -p fixture-maker -- check    # verify the committed payload (non-zero on drift)
```

A change to chunk content or the persist/manifest format changes the committed
hashes; regenerate deliberately and review the diff.

## Consuming the fixture

Decode the committed chunk snapshots with
`rule_voxel_edit::persist::decode_chunk_snapshot` (each reconstructs a `VoxelChunk`
that preserves its `content_hash`), or rebuild the world in memory deterministically
with `fixture_maker::build_world()`.
