# Voxel interaction fixture

Stable generated evidence for the basic graphical voxel interaction proof (#2645).

The committed machine-readable manifest in this directory is generated by
`engine-rs/crates/tools/fixture-maker`; do not hand-edit generated JSON. The
scenario deliberately reuses `harness/fixtures/voxel-world/` when that canonical
fixture is sufficient, rather than creating a parallel terrain shape.

## Contents

| File | Generated by | Purpose |
|---|---|---|
| `basic-selection.json` | `fixture_maker::render_interaction_fixture` | Basic interaction scenario metadata: source terrain fixture, deterministic generation metadata, initial camera, center selection ray, authoritative voxel hit, edit anchor, and intended public `setVoxel` edit. |

## Regenerate / verify

```bash
cd engine-rs
cargo run -p fixture-maker -- write    # regenerates voxel-world and voxel-interaction payloads
cargo run -p fixture-maker -- check    # verifies committed payloads (non-zero on drift)
```

## Stable selection

The current golden records a camera pose whose center/crosshair ray is generated by
`fixture-maker` using the same yaw/pitch forward-vector convention as the runtime
camera basis. That generated center ray selects the top face of voxel `(1, 1, 0)`
in chunk `(0, 0, 0)` with face `posZ`; its edit anchor is `(1, 1, 1)`.
Downstream collision, runtime-meshing, `asha-testing`, and Agora proof tasks should
use the manifest rather than re-deriving a different ad-hoc selection.
