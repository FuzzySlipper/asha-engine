# Material feedback projection

ASHA material feedback is a typed, retained render projection for communicating
gameplay state through tint and emission. Gameplay authority may choose a state
such as normal, warning, or disabled; the renderer values that visualize the
state do not become authority and never feed back into rules.

## Contract

`RenderMaterialDescriptor` schema version 2 carries:

- base linear RGBA colour and optional texture identity;
- roughness and UV strategy;
- a linear RGBA `textureTint`;
- linear RGB `emissionColor` and non-negative `emissionIntensity`.

The runtime decoder accepts unversioned and explicit schema-v1 descriptors for
wire compatibility. It normalizes them to schema v2 with white texture tint,
the base colour as emission colour, and the legacy `emissive` scalar as emission
intensity. Unsupported schema versions and malformed ranges fail closed before
renderer invocation. Rust projection emits only schema v2.

Catalog material style carries the same tint and emission colour. Older stored
catalog JSON that omits those fields decodes with the same neutral defaults, so
existing material assets keep their prior appearance.

## Per-instance feedback

`setMaterialInstanceParameters` addresses one existing static-mesh render handle
and one declared material slot. Its `parameters` value is a complete visual
override containing texture tint, emission colour, and emission intensity.
Supplying `null` clears the override and returns the slot to descriptor defaults.

The retained renderer preserves the render handle and shared mesh geometry. It
creates an instance-owned `MeshStandardMaterial` only for the targeted slot;
unaffected instances continue sharing the asset material. Replacing or clearing
an override disposes the prior instance-owned material. Redefining the base
descriptor rebuilds shared materials and reapplies live instance overrides.

Unknown handles, non-static-mesh handles, and unbound slots are classified
application errors. There is no global registry mutation, arbitrary shader
parameter name, plugin callback, or renderer-to-authority channel.

## Evidence

- `harness/fixtures/render-diffs/material-feedback.json` is generated by the Rust
  scene projector from authoritative scene/catalog state, then receives a typed
  gameplay-presentation warning operation for the second retained instance.
- `harness/goldens/render-diffs/material-feedback.snapshot` shows one normal and
  one tinted/emissive instance with stable handles and a shared asset.
- Runtime decoding tests cover schema-v1 migration, schema-v2 values, invalid
  versions/ranges, and the typed update operation.
- Renderer and neutral-projection tests cover one-instance isolation, unchanged
  geometry/handle identity, reset, disposal, and invalid handle/slot rejection.
- A live Chromium run through `@asha/renderer-host` rendered the committed frame
  in the ASHA demo WebGL surface and asserted the Three.js backend plus distinct
  retained states. This is live evidence, not a committed pixel golden.

## Non-claims

This slice does not add shader graphs, custom shaders, PBR texture maps, atlas
loading, derived material assets, parameter animation, or blending. Texture byte
loading remains a renderer-provider concern. Material values remain projection,
not canonical gameplay state.
